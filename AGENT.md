# [seed] これまでの整理まとめ

## 1. アプリの背景・コンセプト

### 1-1. 問題意識

- ジュニアエンジニアとして  
  **「何か作りたいけど、ToDoアプリみたいな“ポートフォリオ用題材”は面白くない」** というモヤモヤがある。
- 「ポートフォリオのための作品」ではなく、  
  **誰かの“本当に欲しいアイデア”をもとに実践的な開発がしたい**。
- 一方で、こうしたサイトの多くは  
  **アイデア提供側のインセンティブが弱い（わざわざ出すメリットが薄い／本気アイデアは出しづらい）**  
  という課題を抱えがち。

### 1-2. [seed] のコンセプト

- アプリ名：**[seed]**
  - 「アイデアの種（seed）」を誰かが掲げ、それをエンジニアが育ててソリューションにするイメージ。
- 目的：
  - **アイデア提供側**：  
    アイデアの「叩き台」や「検証の場」を得る。
  - **エンジニア側**：  
    就活で使えるレベルの **“実戦ポートフォリオ” を作る場** を得る。
- 将来的な理想：
  - ここで作られたソリューションの  
    **「採用回数」「評価」「利用状況」** を可視化し、就活市場・転職市場でも評価される仕組みにしたい。
  - アイデアそのものにも、閲覧数・インプレッション・採用状況などに応じて  
    **収益が還元される世界**を目指す（広告やマッチング手数料などから分配）。

---

## 2. ユーザー像とインセンティブ

### 2-1. 想定ユーザー

1. **アイデア提供者**
   - エンジニアでも非エンジニアでもよい。
   - 「こういうツール欲しい」「この課題を誰かに解いてほしい」と思っている人。

2. **ビルダー（エンジニア）**
   - 主にジュニア〜中級を想定。
   - 「ToDoアプリではなく、  
     **誰かのリアルな課題を解くプロジェクトをポートフォリオにしたい**」人。

3. **将来の追加ペルソナ**
   - 企業の採用担当・リクルーター。
   - 実務に近い開発実績を見て、候補者を探したい人。

### 2-2. インセンティブの方向性

#### エンジニア側

- 誰かのリアルなニーズを起点としたソリューション開発ができる。
- 将来的にはプロフィールで：
  - 作成したソリューション一覧
  - 各ソリューションの採用回数・評価
  - 使用技術スタック
- などが可視化される → **実践ポートフォリオ**として就活・転職に提示可能。

#### アイデア提供側

- アイデアごとに以下が見えるようにしたい：
  - 閲覧数（どれだけ見られているか）
  - 「作ってみたい」と手を挙げた人数
  - 実際に作られたソリューション数
- → アイデアの **反応・実現可能性の検証**ができる。
- 将来的には、プラットフォームの収益（広告・紹介料など）の一部を  
  **アイデア提供者にも還元**できるようなモデルを想定。

#### 収益イメージ（将来）

- サイト全体の広告収入や、採用・案件成立時の紹介料などを
  - [seed] 運営
  - アイデア提供者
  - ソリューション開発者  
  に分配する方向で検討（現時点では「将来の構想」として整理）。

---

## 3. 現時点の v1 方針

- まずは **「アイデア掲示板サイト」** としてシンプルにリリースする。
- **収益化機能はリリース後に段階的に追加する前提**。
- v1 で実装する範囲（フロント視点）：
  - アイデアの投稿
  - アイデアの一覧表示（掲示板）
  - アイデアを実現したソリューションの投稿
  - マイページ／他ユーザーのプロフィール表示
  - Googleアカウントによるログイン（Firebase Auth）

- 現フェーズの方針：
  - DB・バックエンド設計は一旦置いておき、  
    **まずはフロントエンド・画面構成から作り始める**。

---

## 4. 技術スタックの方向性

- フロントエンド：**TypeScript + Angular**
- 認証：**Firebase Authentication（まずは Google ログインのみ）**
- DB：**PostgreSQL**（ただし今は触らず、後から API と一緒に設計）
- バックエンド候補（将来）：
  - 本線案：**Java + Spring Boot + PostgreSQL + Firebase Auth 連携**
    - Firebase Auth の IDトークンを Spring Boot 側で検証し、  
      アプリ内ユーザー情報と紐づける構成。
- 現時点は：
  - **Angular で画面とルーティング → ダミーデータで UI を固める → その後 API / DB を接続**  
    という順で進める方針。

## 5. 画面構成（[seed] v1）

### 5-1. 共通ヘッダー（全ページ共通）

- 表示位置：画面上部（固定ヘッダ）

#### 左側

- （将来）サービスアイコン
- サービス名：**[seed]**
- 現在のページタイトル
  - 例：掲示板 / アイデア作成 / ソリューション投稿 / マイページ など

#### 右側

- 通知ボタン（ベルアイコン）  
  - v1 では表示のみでも可
- 未ログイン時：
  - 「ログイン」ボタン（`/login` へ）
- ログイン時：
  - 自分のアカウントアイコン（丸アイコン）
  - クリックでメニュー表示
    - マイページ (`/me`)
    - ログアウト

---

### 5-2. 掲示板画面（アイデア一覧）

- URL：`/`
- 役割：
  - 投稿された**アイデア一覧**を閲覧する。
  - 各アイデアに対するソリューション数や投稿者情報も確認。
  - アイデア投稿、ソリューション投稿、ユーザーページへの入口。

#### 主な要素

1. ホームページ
   - タイトル とコンセプト紹介文、その下にはアイデア掲示板のデモ(?)的な紹介用の表示
   - タイトル例：``
   - 説明：  
     「[seed] は、アイデアを掲げて、実際に作ってみるための場です。」など。

2. 掲示板
   - アイデアの一覧が表示される掲示板

3. アイデア詳細ページ
   - アイデアの詳細とそれに対してコメントできるような画面。
    
4. アイデア投稿ボタン
   - `+ アイデアを投稿する`
   - クリックで `/ideas/new` へ
   - 未ログイン時は `/login` に誘導 or ログインを促す

5. フィルタ／ソートバー
   - カテゴリ（Webアプリ / モバイル / ブラウザ拡張 等）
   - 難易度（★1〜5）
   - ステータス（募集中 / 開発中 / ソリューションあり 等）
   - 並び替え（新着順 / 人気順）(デフォルトは人気順)
   - 検索機能

6. アイデアカード一覧
   - カードごとに表示する項目：
     - カテゴリ
     - 難易度（★表示）
     - ステータス（募集中 / 開発中 / ソリューションあり）
     - タイトル
     - 一行サマリ
     - 投稿者（アイコン＋アカウント名）
       - クリックで「簡易プロフィールポップ」（後述）
       - さらにそこからユーザーページへリンク
     - ソリューション数
     - 将来的には「作ってみたい」人数など
     - 最終更新日時
     - ボタン：
       - `詳細を見る`（将来の詳細画面 or モーダル）
       - `このアイデアでソリューション投稿` → `/solutions/new?ideaId=xxx`

7. ページネーション
   - 前へ / 次へ
   - ページ番号表示

### 5-3. アイデア作成画面

- URL：`/ideas/new`
- 役割：
  - アイデア提供者が [seed] に「アイデアの種」を投稿する。

#### 入力項目（v1）

- タイトル（必須）
- 一行サマリ（必須）
- 課題（Problem）（必須）
- 解決のイメージ（Solution）（必須 or 任意は今後検討）
- カテゴリ（必須）
- 公開レベル（必須）
  - オープン
  - 課題のみ公開（Problemのみ表示、Solution は将来的に制御）
- 難易度（★1〜5）（必須）
- 想定プラットフォーム（任意）
  - 例：Web / SPA / Chrome拡張 等
- 連絡先リンク（任意）
  - Twitter / Discord / メールなどのURL

- ボタン：
  - キャンセル（掲示板へ戻る）
  - 投稿する

---

### 5-4. ソリューション投稿画面

- URL：`/solutions/new?ideaId=xxx`
- 役割：
  - エンジニアが「アイデアを実現してみたもの（ソリューション）」を投稿する。

#### 要素

- 対象アイデア（読み取り専用）
  - タイトル
  - （可能なら一行サマリ）
  - 「アイデア詳細を見る」リンク（将来の詳細画面 or モーダルを想定）

- 入力項目：
  - ソリューション名（任意、アプリ名など）
  - 何を作ったか（説明）
  - リンク（必須）
    - デモURL or GitHubリポジトリ等
  - 使用技術（任意）
    - 例：Angular / Spring Boot / PostgreSQL

- ボタン：
  - キャンセル
  - 投稿する

---

### 5-5. マイページ

- URL：`/me`
- 役割：
  - 自分のプロフィールと、自分の投稿したアイデア・ソリューションを一覧。

#### プロフィール項目

- アイコン
- アカウント名
- ID
- プロフィール文
- 外部リンク：
  - Twitter(X)
  - GitHub
  - ポートフォリオサイト など
- 「プロフィールを編集」ボタン（将来的にフォームを表示）

#### タブ構成（案）

- 「アイデア」タブ
  - 自分が投稿したアイデア一覧（簡易カード）
- 「ソリューション」タブ
  - 自分が投稿したソリューション一覧
  - 各ソリューションに：
    - 対象アイデアタイトル
    - ソリューション名
    - 投稿日時
    - リンク

---

### 5-6. 他ユーザーアカウントページ

- URL：`/users/:id`
- 役割：
  - 他ユーザーのプロフィールと、そのユーザーのアイデア・ソリューションを閲覧。

#### 内容

- マイページの閲覧専用版：
  - アイコン
  - アカウント名
  - ID
  - プロフィール文
  - Twitter/GitHub/ポートフォリオリンク
  - 「アイデア」「ソリューション」タブで投稿一覧

#### 簡易プロフィールポップ

- 掲示板などで投稿者のアイコン/名前をクリックすると表示。
- 内容例：
  - アイコン
  - アカウント名
  - 一言プロフィール
  - Twitter/GitHubリンク
  - 「このユーザーのページへ」リンク（`/users/:id`）

---

### 5-7. ログイン画面

- URL：`/login`
- 役割：
  - Google アカウントでログインするための導線。

#### 要素

- タイトル：`ログイン`
- 説明文：
  - [seed] で
    - アイデア投稿
    - ソリューション投稿
    - マイページ作成
    ができるようになることを案内。
- ボタン：
  - `Googleでログイン`
    - クリックで Firebase Auth の Google Sign-In を実行
- 注意書き：
  - 初
